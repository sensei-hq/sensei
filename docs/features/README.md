# Features

User-facing feature specifications. Each describes what the user experiences, not how it's built. Scenarios use Gherkin format for testable acceptance criteria.

| 14 | [Guardrails & Context](./14-guardrails-context.md) | Living guardrails, refocus commands, context preservation across compaction |
| 15 | [Pattern Enforcement](./15-pattern-enforcement.md) | Detect patterns, enforce during build, catch violations in review, present architectural options |

| 17 | [Quality & Coaching](./17-quality-coaching.md) | Event capture, FTR/turn metrics, dashboard visualization, guided coaching prompts |

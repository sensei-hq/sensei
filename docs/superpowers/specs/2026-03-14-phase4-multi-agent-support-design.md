# Phase 4: Multi-Agent Support — Design Spec

## Goal

`sensei setup --agent claude` generates four project-specific skill files and installs them into `~/.claude/skills/`. Skills are generated by Claude (via `ClaudeBackend`) from a `ProjectProfile` derived from the indexed codebase, then validated by Claude before writing. Skill status is visible in a dashboard Agent Skills view.

---

## Problem

Sensei already indexes the codebase and tracks sessions, but agents starting work on a repo have no project-specific skills to orient them. Skills written by hand are generic. Phase 4 generates project-specific skills automatically — actual package names, actual CLI commands, actual patterns — and validates them so agents get accurate guidance on first load.

---

## Architecture

```
sensei setup --agent claude
  → extractProjectProfile(db, repoId, repoPath)    reads symbols + config
  → ClaudeBackend.generate()                        4 prompts, one per category
  → SkillValidator.validate()                       Claude reviews each for accuracy
  → ClaudeAdapter.writeSkills()                     writes to ~/.claude/skills/
  → .sensei/agent-skills.json                       metadata (paths, timestamps)

install_skills MCP tool
  → same pipeline, triggered from active agent session

Dashboard /repos/[id]/agents
  → reads .sensei/agent-skills.json                 skill status (present/stale/missing)
  → Regenerate button                               form action spawns sensei setup --agent claude
```

Skill metadata stored in `.sensei/agent-skills.json` — no new DB migration required.

## Types (`packages/shared/src/types.ts`)

```typescript
export interface ProjectProfile {
  repoName: string;
  repoPath: string;
  dominantLanguage: string;             // 'typescript' | 'python' | etc.
  framework: string | null;             // 'sveltekit' | 'react' | 'express' | null
  packageNames: string[];               // monorepo packages e.g. ['engine', 'cli', 'shared']
  keySymbols: string[];                 // top 20 most-referenced exported symbols
  testPattern: string;                  // e.g. '*.spec.ts'
  cliCommands: Record<string, string>;  // from package.json scripts
  senseiConfig: string;                 // serialised .sensei/config.yaml summary
}

export interface AgentSkillFile {
  category: 'orientation' | 'workflow' | 'context' | 'patterns';
  path: string;          // absolute path to written skill file
  generatedAt: string;   // ISO timestamp
}

export interface AgentSkillsManifest {
  agent: 'claude';
  repoSlug: string;
  skills: AgentSkillFile[];
  updatedAt: string;
}
```

### `extractProjectProfile`

```typescript
export async function extractProjectProfile(
  db: SupabaseClient,
  repoId: string,
  repoPath: string
): Promise<ProjectProfile>
```

- Queries `repos` table for `name`
- Queries `symbols` for top 20 by reference count (or alphabetical fallback), and file extension distribution for `dominantLanguage`
- Reads `package.json` from `repoPath`: extracts `scripts` → `cliCommands`, `dependencies`+`devDependencies` → `framework` detection, `workspaces` → `packageNames`
- Reads `.sensei/config.yaml` for `senseiConfig` summary
- Throws on DB error; throws if `package.json` missing

### `SkillGenerator`

```typescript
export class SkillGenerator {
  constructor(private model: ModelBackend, private profile: ProjectProfile) {}

  async generate(): Promise<Record<'orientation' | 'workflow' | 'context' | 'patterns', string>>
}
```

Four prompts, each injecting relevant `ProjectProfile` fields:

| Category | Profile fields used |
|----------|-------------------|
| `orientation` | repoName, packageNames, keySymbols, dominantLanguage, framework |
| `workflow` | cliCommands, testPattern, senseiConfig |
| `context` | keySymbols, packageNames, senseiConfig |
| `patterns` | dominantLanguage, framework, testPattern, cliCommands |

Each prompt instructs Claude to write a valid skill in SKILL.md format with correct YAML frontmatter.

### `SkillValidator`

```typescript
export class SkillValidator {
  constructor(private model: ModelBackend, private profile: ProjectProfile) {}

  async validate(category: string, skillMarkdown: string): Promise<{ valid: boolean; issues: string[] }>
}
```

Sends review prompt: *"Check this skill for accuracy against the project profile. Verify: package names match, commands are real, no hallucinated file paths. Reply VALID or list issues."*

`SkillGenerator` calls `validate()` after each generation; on invalid result, retries with issues appended to the prompt. Max 2 retries per skill. If still invalid after 2 retries, throws with the remaining issues listed.

### `AgentAdapter` interface

```typescript
export interface AgentAdapter {
  skillsDir: string;
  writeSkills(
    skills: Record<string, string>,
    repoSlug: string
  ): Promise<AgentSkillFile[]>;
  installedSkills(repoSlug: string): Promise<AgentSkillFile[]>;
}
```

### `ClaudeAdapter`

- `skillsDir`: `~/.claude/skills/`
- `writeSkills`: writes `sensei-{repoSlug}-{category}.md` for each category; creates dir if missing; returns `AgentSkillFile[]`
- `installedSkills`: reads dir, filters by `sensei-{repoSlug}-` prefix, returns list with mtime as `generatedAt`

## CLI Changes (`packages/cli/src/commands/setup.ts`)

New `setupAgent(repoPath: string, agent: 'claude'): Promise<void>` function:

```
1. intro("sensei setup --agent claude")
2. makeSenseiClient(repoPath) + loadSenseiConfig(repoPath) — throws if not initialised
3. extractProjectProfile(db, repoId, repoPath)
4. new ClaudeBackend() → backend.init() — fails fast if ANTHROPIC_API_KEY missing
5. new SkillGenerator(backend, profile).generate()
   — spinner: "Generating {category} skill..." × 4
6. new SkillValidator(backend, profile).validate() per skill (within generator loop)
7. new ClaudeAdapter().writeSkills(skills, repoSlug)
8. write .sensei/agent-skills.json (AgentSkillsManifest)
9. outro: "4 skill files written to ~/.claude/skills/"
```

`cli.ts` gains `--agent <agent>` option on the `setup` subcommand. Accepted value: `claude` (others return a "not yet supported" message).

## Error Handling

| Operation | On error |
|-----------|----------|
| `extractProjectProfile` | Throws — setup fails clearly |
| `ClaudeBackend.init()` | Throws — no `ANTHROPIC_API_KEY` message |
| `SkillGenerator.generate()` | Throws after 2 retries with validator issues listed |
| `ClaudeAdapter.writeSkills()` | Throws — filesystem error surfaced |
| `installSkillsTool` | Returns `errors[]` — never throws |
| Dashboard load | Returns empty state on missing manifest |

## Test Scope

**`project-profile.spec.ts`**:
- `extractProjectProfile` with fixture symbols returns correct `dominantLanguage` and `keySymbols`
- Missing `package.json` throws

**`skill-generator.spec.ts`**:
- `generate()` with mock `ModelBackend` calls `generate()` exactly 4 times
- Retries on invalid validation result (up to 2 retries)
- Throws after 2 failed retries

**`skill-validator.spec.ts`**:
- Returns `{ valid: true }` when model returns "VALID"
- Returns `{ valid: false, issues: [...] }` when model returns issues list

**`claude-adapter.spec.ts`**:
- `writeSkills()` creates correct filenames in a temp dir
- `installedSkills()` returns only files matching the repo slug prefix

**`claude-backend.spec.ts`** (in `packages/server`):
- `init()` throws when `ANTHROPIC_API_KEY` absent from env
- `embed()` throws `NotImplementedError`

## Done When

- `sensei setup --agent claude` runs on the sensei repo → 4 skill files written to `~/.claude/skills/` containing sensei-specific content (e.g. references to `packages/engine`, `bun run test`)
- Each skill validated by Claude before writing — invalid skills cause a clear error with listed issues
- `install_skills` MCP tool produces the same result from within an agent session
- Dashboard `/repos/[id]/agents` shows all 4 skill files with status badges
- "Regenerate Skills" button re-runs the pipeline and updates the view
